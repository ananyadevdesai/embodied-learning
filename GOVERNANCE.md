# Governance

## Maintainer

The project is maintained by [your name/username]. Decisions are made transparently and explained publicly.

## Decision Making

- **Pull Requests**: Reviewed weekly (typically within 7 days)
- **Issues**: Triaged and labeled regularly
- **Decisions**: Explained in PR comments, issue threads, or discussions
- **Forks**: Encouraged - if you want to take the project in a different direction, fork it

## Contribution Review

- All PRs are welcome and will be reviewed
- Feedback is provided constructively
- Rejected PRs include clear explanations
- Maintainers aim to help contributors succeed

## Project Direction

- Major changes are discussed in GitHub Discussions
- Schema changes require backward compatibility or clear migration paths
- New features should align with core principles (see README)

## Transparency

- All decisions are public
- Rationale is documented
- Disagreements are resolved through discussion

## Questions?

Open a Discussion or Issue. We're here to help.
